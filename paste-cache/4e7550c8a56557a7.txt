## 2. Technical Architecture & Core Layers

### Layer 1: Layout Data Engine (JSON-First)
- **Data Format:** Layouts are stored strictly as structured JSON in the database (`Page -> Sections -> Rows -> Boxes`). Allowed Box types: `Text, Image, Video, Button, Spacer`. Styling parameters (spacing, overlays 0-100%) must be saved as parameters. NO inline HTML/CSS injection into the JSON structure.

### Layer 2: Internationalization (i18n) & Domain Agnosticism
- **i18n Layers:** Separate UI strings (using `t('key')` helper matching `/lang/de.php`) from Content translations (dynamic database columns like `_en`).
- **Domain Agnosticism:** All media paths and internal links MUST be stored as relative paths (e.g., `/uploads/...`). Absolute URLs containing the domain are banned in the DB layer.
- **Tenant Isolation:** Enforce routing via a central Master Registry. Tenant configuration and folder structures must be bound to a unique, immutable Tenant-ID, never to the active domain string (e.g., `/storage/tenants/{tenant_id}/uploads/`).

---

## 3. Automated Compliance & Design Guardrails
- **DSGVO Enforcement:** External asset calls are completely banned. Fonts must be served locally. Video embeds must utilize a privacy-compliant 2-click solution in Vanilla JS.
- **Data Privacy:** Frontend forms must implement CSRF validation, a hidden honeypot, and strict retention periods with automated deletion cron-hooks. The image pipeline must automatically strip all EXIF/metadata via PHP before saving to disk.
- **Anti-AI-Slop Styling:** Clean, high-end, reduced corporate design suited for professional German businesses. Absolutely NO neon gradients or glowing tech aesthetics. Use pure vector SVGs.
